# 05 — Definitions

---

# Overview

This document defines the official business terminology used throughout the Project Business Object.

These definitions establish a shared enterprise vocabulary that ensures Project Managers, Executives, Project Management Offices, Artificial Intelligence and enterprise systems interpret project information consistently.

These definitions are the authoritative terminology for project governance within Swissbay Nexus.

---

# Core Definitions

## Project

A temporary, governed initiative undertaken to deliver one or more defined business outcomes within agreed scope, time, cost and quality constraints.

Every Project is uniquely represented by one Project Business Object.

---

## Project Identifier

The permanent, globally unique identifier assigned to every Project.

Example

```
PRJ-000145
```

Project Identifiers:

- never change
- are never reused
- uniquely identify a Project

---

## Project Name

The approved business name describing the Project.

Example

```
Copper Mine Expansion Phase 2
```

---

## Project Sponsor

The executive responsible for funding, strategic alignment and major governance decisions.

Sponsors remain accountable for business success.

---

## Project Manager

The individual responsible for day-to-day planning, coordination, execution and delivery.

Project Managers manage execution but do not necessarily own the business outcome.

---

## Business Owner

The executive accountable for ensuring that the project delivers its intended business value.

---

## Scope

The agreed boundaries defining what the Project will and will not deliver.

Scope changes require formal governance.

---

## Deliverable

A measurable output produced by the Project.

Examples include:

- Software System
- Construction Phase
- Process Improvement
- Technical Report
- Infrastructure Upgrade

---

## Milestone

A significant point in the Project schedule representing measurable progress.

Examples include:

- Planning Approved
- Construction Complete
- User Acceptance Testing Complete
- Go-Live
- Project Closure

---

## Task

A defined unit of work required to complete a deliverable.

Tasks support project execution but do not replace the Project Business Object.

---

## Dependency

A relationship where one activity relies upon another before it can begin or complete.

Dependencies influence scheduling and planning.

---

## Risk

An uncertain event that may positively or negatively affect project objectives.

Risks are monitored throughout the project lifecycle.

---

## Issue

A problem that has already occurred and requires action.

Issues differ from risks because they are active events.

---

## Assumption

A condition accepted as true for planning purposes until proven otherwise.

Assumptions should be documented and reviewed.

---

## Change Request

A formal proposal to modify project scope, schedule, budget or deliverables.

All Change Requests require governed approval.

---

## Budget

The approved financial allocation for the Project.

Budget changes follow formal governance.

---

## Baseline

The approved reference point against which project performance is measured.

Projects typically maintain:

- Scope Baseline
- Schedule Baseline
- Cost Baseline

---

## Stage Gate

A formal governance checkpoint where project continuation is reviewed and approved.

Projects should not proceed beyond a Stage Gate without approval.

---

## Lessons Learned

Knowledge captured during or after project execution to improve future projects.

Lessons become organisational assets.

---

## Artificial Intelligence

Artificial Intelligence refers to governed capabilities that analyse Project information to provide forecasts, recommendations and insights.

AI supports project delivery but never replaces human governance.

# Terminology Rules

The following terminology applies throughout the Project Business Object:

- Use **Project Identifier** rather than Project ID in formal documentation.
- Use **Project Sponsor** for executive ownership.
- Use **Business Owner** for business accountability.
- Use **Project Manager** for operational delivery.
- Use **Deliverable** rather than Output where practical.
- Use **Milestone** only for significant governance events.

---

# AI Interpretation

Artificial Intelligence must interpret Project terminology according to this document.

Where ambiguity exists, these definitions take precedence over external interpretations.

Changes require:

- PMO Review
- Enterprise Architecture Review
- Governance Approval
- Version Update

This document establishes the official enterprise vocabulary for the Project Business Object.
