# 06 — Project Lifecycle

---

# Overview

The Project Lifecycle defines the governed progression of every Project within Swissbay Nexus.

Every Project follows a standard lifecycle that ensures consistent governance, executive oversight, controlled investment and measurable delivery outcomes regardless of methodology or business unit.

A Project exists in one—and only one—lifecycle stage at any given time.

---

# Purpose

The Project Lifecycle exists to:

- standardise project delivery
- strengthen governance
- improve executive visibility
- improve planning
- reduce project risk
- enable automation
- support Artificial Intelligence
- improve delivery consistency

The lifecycle provides one operating model for enterprise projects.

---

# Lifecycle Overview

Every Project progresses through the following stages.

```text
Idea

↓

Business Case

↓

Initiation

↓

Planning

↓

Execution

↓

Monitoring & Control

↓

Stage Gate Review

↓

Project Closure

↓

Archived
```

Projects may loop between Planning, Execution and Monitoring multiple times depending on governance requirements.

---

# Stage 1 — Idea

## Purpose

Capture an initial concept or opportunity.

### Activities

- identify business need
- capture opportunity
- assign sponsor
- record strategic objective

### Exit Criteria

Business Case approved for development.

---

# Stage 2 — Business Case

## Purpose

Determine whether the proposed Project should proceed.

### Activities

- cost estimation
- benefit analysis
- feasibility assessment
- strategic alignment
- preliminary risk assessment

### Outputs

- Approved Business Case
- Rejected Proposal

---

# Stage 3 — Initiation

## Purpose

Officially establish the Project.

### Activities

- assign Project Manager
- define objectives
- identify stakeholders
- allocate initial budget
- create Project record

### Outputs

Approved Project Charter.

---

# Stage 4 — Planning

## Purpose

Prepare the Project for execution.

### Activities

- define scope
- create schedule
- define milestones
- allocate resources
- identify risks
- estimate costs
- establish governance

### Outputs

Approved Project Management Plan.

---

# Stage 5 — Execution

## Purpose

Perform the planned work.

### Activities

- coordinate teams
- produce deliverables
- manage contractors
- consume budget
- update progress
- manage communications

Execution produces measurable business outcomes.

---

# Stage 6 — Monitoring & Control

## Purpose

Measure performance and maintain governance.

### Activities

- monitor schedule
- monitor budget
- monitor scope
- monitor risks
- manage issues
- approve change requests
- report project health

Monitoring occurs continuously throughout execution.

---

# Stage 7 — Stage Gate Review

## Purpose

Review Project performance before continuing.

### Activities

- executive review
- financial review
- risk review
- quality review
- benefit review

### Outcomes

- Continue
- Pause
- Replan
- Cancel

---

# Stage 8 — Project Closure

## Purpose

Formally complete the Project.

### Activities

- final acceptance
- financial close
- resource release
- document lessons learned
- archive project information

### Outputs

Completed Project.

---

# Stage 9 — Archived

## Purpose

Retain Project history.

Archived Projects:

- remain searchable
- support reporting
- support AI
- support audits
- cannot be modified without governance approval

---

# Lifecycle Events

Major lifecycle events include:

- Project Proposed
- Business Case Approved
- Project Initiated
- Planning Approved
- Execution Started
- Milestone Achieved
- Risk Escalated
- Stage Gate Passed
- Project Closed
- Project Archived

These events trigger workflows, dashboards and automation.

---

# AI Throughout the Lifecycle

Artificial Intelligence may:

- estimate project duration
- predict delivery risk
- recommend resource allocation
- forecast budget variance
- analyse project health
- recommend corrective actions

AI remains advisory throughout every lifecycle stage.

---

# Governance

Every lifecycle transition requires:

- validation
- approval
- audit logging
- workflow execution
- Business Rule compliance

The Swissbay Governance Engine enforces lifecycle integrity.

---

# Lifecycle Summary

The Project Lifecycle establishes the governed progression of every Project from initial idea through closure and archival.

By defining controlled lifecycle stages, approvals, governance checkpoints and AI-assisted monitoring, Swissbay ensures predictable project delivery, improved executive oversight and consistent organisational execution.
