# 14 — Key Performance Indicators (KPIs)

---

# Overview

The Project KPI Model defines the official performance measurements used to evaluate projects throughout their lifecycle within Swissbay Nexus.

KPIs provide consistent, governed metrics for measuring project health, delivery performance, financial control, governance compliance and business outcomes.

Every KPI follows the Swissbay KPI Standard (KPI-000).

---

# Purpose

The Project KPI Model exists to:

- measure project performance
- improve delivery predictability
- strengthen governance
- improve executive visibility
- optimise resource utilisation
- support Artificial Intelligence
- measure strategic value

KPIs measure business outcomes rather than project activity.

# KPI-PROJ-001 — Project Health Score

## Purpose

Provides an overall assessment of project health.

### Inputs

- Schedule Performance
- Cost Performance
- Risk Level
- Issue Count
- Milestone Progress
- Deliverable Status

### Frequency

Daily

### Audience

- Project Managers
- PMO
- Executive

---

# KPI-PROJ-002 — Schedule Performance Index (SPI)

## Purpose

Measures schedule efficiency.

### Formula

```
Earned Value

÷

Planned Value
```

### Interpretation

- >1.0 = Ahead of Schedule
- 1.0 = On Schedule
- <1.0 = Behind Schedule

---

# KPI-PROJ-003 — Cost Performance Index (CPI)

## Purpose

Measures budget efficiency.

### Formula

```
Earned Value

÷

Actual Cost
```

### Interpretation

- >1.0 = Under Budget
- 1.0 = On Budget
- <1.0 = Over Budget

---

# KPI-PROJ-004 — Budget Variance

## Formula

```
Approved Budget

-

Actual Cost
```

Measures financial performance throughout execution.

---

# KPI-PROJ-005 — Milestone Completion Rate

## Formula

```
Completed Milestones

÷

Planned Milestones

×

100
```

Tracks delivery progress.

---

# KPI-PROJ-006 — Deliverable Acceptance Rate

## Formula

```
Accepted Deliverables

÷

Completed Deliverables

×

100
```

Measures quality and customer acceptance.

---

# KPI-PROJ-007 — Resource Utilisation

## Formula

```
Productive Hours

÷

Available Hours

×

100
```

Supports workforce optimisation.

---

# KPI-PROJ-008 — Risk Resolution Rate

## Formula

```
Closed Risks

÷

Total Risks

×

100
```

Measures governance effectiveness.

---

# KPI-PROJ-009 — Change Request Cycle Time

Measures the average time taken to approve or reject Change Requests.

Lower values improve project agility while maintaining governance.

---

# KPI-PROJ-010 — Customer Satisfaction

Measures stakeholder satisfaction after delivery.

Inputs may include:

- surveys
- acceptance reviews
- project evaluations

- calculation defined
- owner assigned
- reporting frequency defined
