# 10 — Data Quality

## Purpose

Data quality determines whether Nexus can be trusted.

## Data Quality Dimensions

| Dimension | Meaning |
|---|---|
| Completeness | Required fields are filled |
| Accuracy | Data reflects reality |
| Consistency | Same rules used everywhere |
| Timeliness | Data is up to date |
| Ownership | Someone is responsible |
| Traceability | Source is known |

## Quality Rules

- Important data must have an owner.
- Required fields must be defined in specifications.
- Critical fields must be reviewable.
- AI-generated data must be distinguishable from approved business data.

## Example

A customer without a contact person is incomplete.

A quote without a customer ID is invalid.

An invoice without an order reference is high risk.
