# 11 — Data Ownership

## Purpose

Data ownership defines who is responsible for maintaining information.

## Ownership Rule

Every important data domain must have an owner.

## Example Ownership

| Data Area | Owner |
|---|---|
| Customer details | Sales / CRM owner |
| Payment terms | Finance |
| Supplier pricing | Procurement |
| Product catalogue | Procurement |
| Invoice data | Finance |
| Delivery address | Operations / Sales |
| AI-generated notes | Knowledge Curator |

## Responsibilities

Data owners are responsible for:

- accuracy
- completeness
- review frequency
- change approval
- resolving conflicts
- ensuring data supports workflows

## Rule

If nobody owns a data field, it will eventually become unreliable.
